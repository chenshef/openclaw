## Lessons Learned
- Long exec jobs in main session freeze Telegram — always use isolated cron/subagent for pentest scans
- Subagents can return "completed" without actually producing output — verify files exist before reporting success
- nuclei/subfinder don't return --version but work fine in practice
- When pentest job fails silently: check results dir for new folders, re-run if nothing found
- FTP brute-force results came back via async exec — stored internally, not relayed
- WordPress REST API may be blocked but XML-RPC often remains open (different attack vector)
